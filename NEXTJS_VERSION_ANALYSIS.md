## Key Differences Between Next.js 15.1.0 and 16.1.1

### 1. **Async Request APIs (Breaking Change in Next.js 15+)**

Your codebase has **already adopted** this pattern correctly! ✅

#### Current Implementation (Correct for Next.js 15+)
```typescript
// Page Components
interface DashboardPageProps {
  params: Promise<{
    storeId: string;
  }>
}

const DashboardPage = async ({ params }: DashboardPageProps) => {
  const { storeId } = await params; // ✅ Awaiting params
  // ...
}

// API Routes
export async function PATCH(
  req: NextRequest,
  { params }: { params: Promise<{ storeId: string }> }
) {
  const { storeId } = await params; // ✅ Awaiting params
  // ...
}

// Layouts
export default async function DashboardLayout({
  children,
  params,
}: {
  children: React.ReactNode;
  params: Promise<{ storeId: string }>;
}) {
  const { storeId } = await params; // ✅ Awaiting params
  // ...
}
```

**Why This Changed:**
- Next.js 15 made `params` and `searchParams` asynchronous to support dynamic rendering optimizations
- This is a **breaking change** from Next.js 14 where params were synchronous
- Your codebase has correctly implemented this pattern

#### Old Pattern (Next.js 14 and earlier)
```typescript
// ❌ This would NOT work in Next.js 15+
const DashboardPage = async ({ params }: DashboardPageProps) => {
  const { storeId } = params; // Direct access (no await)
}
```
